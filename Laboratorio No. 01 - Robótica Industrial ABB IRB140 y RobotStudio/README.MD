## DescripciГіn de la SoluciГіn

El proyecto consiste en la **decoraciГіn automatizada de una torta virtual** usando un robot industrial ABB IRB 140 programado en RAPID. El robot, equipado con un marcador fijado al flanche junto con una herramienta diseГұada por los integrantes del equipo, traza los nombres de los integrantes del grupo letra por letra sobre una superficie plana, usando trayectorias continuas y controladas mediante seГұales digitales de entrada y salida.

### Flujo general del sistema

1. El sistema espera una **seГұal digital de entrada (DI_01 = 1)** para iniciar la rutina de decorado.
2. El robot ejecuta secuencialmente las trayectorias de cada letra: **J, U, L, I, A, N** (primer nombre) y **F, E, L, I, P, E** (segundo nombre) y luego una figura, en este caso una cara feliz, regresando a la posiciГіn `Home_b` entre cada letra.
3. Al finalizar, se activa la **seГұal de salida (DI_02 = 1)** y el robot espera confirmaciГіn para terminar.
4. Una segunda seГұal controla el **transporte del pastel** mediante una banda transportadora.

### Restricciones cumplidas

- Velocidades entre **v100 y v200** (dentro del rango 100вҖ“1000)
- Zona de error mГЎxima **z10** (en posiciones de transiciГіn) y **z1** (en trazado)
- Movimiento parte y finaliza desde la posiciГіn **Home**
- Cada nombre escrito como trazos continuos con retorno a `Home_b` entre letras
- Trayectorias definidas sobre el **workobject `julian_wobj`** (torta virtual)

## Workobject

Se definiГі el workobject `julian_wobj` que ubica la torta virtual sobre la superficie de trabajo:

```rapid
TASK PERS wobjdata julian_wobj := [
  FALSE, TRUE, "",
  [[0, 0, 0], [1, 0, 0, 0]],
  [[-653, 300, 382], [0.00753222, 0.999657, 0.000022491, -0.0250846]]
];
```

Las trayectorias de escritura se definen en coordenadas **locales del workobject** (plano Z=0), lo que permite reutilizarlas fГЎcilmente en otro workobject para el cuadrante x(+), y(вҲ’).

## Entradas y Salidas Digitales

| SeГұal | Tipo | FunciГіn |
|-------|------|---------|
| `DO_01` | Entrada digital | Inicia la rutina de decorado |
| `DO_02` | Entrada digital | Activa pose de mantenimiento |
| `DI_02` | Salida digital | Indica que el decorado finalizГі / enciende luz |
| `DI_01` | Salida digital | Controla la banda transportadora |

### LГіgica de control (resumen)

```
Esperar DO_01 = 1
  вҶ’ Ejecutar trayectorias de decorado
  вҶ’ Regresar a Home_b
  вҶ’ Activar DI_02 = 1 (luz encendida)
  вҶ’ Esperar DO_01 = 0

Si DI_02 = 1:
  вҶ’ Apagar DI_01
  вҶ’ Esperar DO_02 = 1 (seГұal para mantenimiento)
```

## DescripciГіn de Funciones RAPID Utilizadas

### Instrucciones de movimiento

| FunciГіn | DescripciГіn |
|---------|-------------|
| `MoveJ` | Movimiento en espacio articular (joint). Usado para desplazamientos grandes sin restricciГіn de trayectoria cartesiana (ej. ir a `Home_b`). |
| `MoveL` | Movimiento lineal en espacio cartesiano. Usado para trazar los segmentos rectos de cada letra. |
| `MoveC` | Movimiento circular en espacio cartesiano. Requiere un punto intermedio y un punto final. Usado para las curvas de letras como **J**, **U**, **P**, **C**, **B**. |
| `MoveAbsJ` | Movimiento a posiciГіn articular absoluta. Гҡtil para ir a la posiciГіn home con todos los ГЎngulos en 0В°. |

### Control de flujo

| FunciГіn | DescripciГіn |
|---------|-------------|
| `WHILE TRUE DO ... ENDWHILE` | Bucle infinito principal del programa. El robot siempre queda en espera de una seГұal. |
| `IF ... THEN ... ENDIF` | Condicional para verificar el estado de las seГұales digitales antes de ejecutar rutinas. |
| `WaitDI` | Pausa la ejecuciГіn hasta que una seГұal digital de entrada alcance el valor especificado. |
| `WaitTime` | Pausa la ejecuciГіn por un tiempo determinado (en segundos). |

### Entradas y salidas digitales

| FunciГіn | DescripciГіn |
|---------|-------------|
| `SetDO` | Establece el valor (0 o 1) de una seГұal digital de salida. |
| `WaitDI` | Espera hasta que una seГұal digital de entrada tome el valor especificado. |

### Tipos de datos usados

| Tipo | DescripciГіn |
|------|-------------|
| `robtarget` | Define una posiciГіn y orientaciГіn del TCP en el espacio. Incluye posiciГіn [x,y,z], cuaterniГіn de orientaciГіn, configuraciГіn del robot y ejes externos. |
| `wobjdata` | Define un objeto de trabajo (sistema de coordenadas de la pieza). Permite reutilizar trayectorias en distintas ubicaciones. |
| `tooldata` | Define los datos de la herramienta: posiciГіn del TCP y masa/inercia. |

## Letras y Trayectorias

### Primer nombre вҖ” JULIAN

| Letra | Procedimientos | Tipo de trayectoria |
|-------|---------------|-------------------|
| J | `Path_J_1`, `Path_J_2` | LГӯnea + arco (MoveL + MoveC) |
| U | `Path_U_1`, `Path_U_2`, `Path_U_3` | LГӯneas + arco inferior |
| L | `Path_L_1`, `Path_L_2` | Dos lГӯneas (vertical + horizontal) |
| I | `Path_I` | LГӯnea horizontal simple |
| A | `Path_A_1`, `Path_A_2`, `Path_A_3` | Dos diagonales + travesaГұo |
| N | `Path_N_1`, `Path_N_2`, `Path_N_3` | Tres segmentos (dos verticales + diagonal) |

### Segundo nombre вҖ” FELIPE 

| Letra | Procedimientos | Tipo de trayectoria |
|-------|---------------|-------------------|
| F | `Path_F_1`, `Path_F_2`, `Path_F_3` | LГӯneas (vertical + horizontales) |
| E | `Path_E_1`вҖ“`Path_E_4` | LГӯneas (vertical + tres horizontales) |
| L | `Path_L_11`, `Path_L_12` | Dos lГӯneas |
| I | `Path_I_11` | LГӯnea simple |

| E | `Path_E_11`вҖ“`Path_E_14` | LГӯneas (vertical + tres horizontales) |

### Dibujo вҖ” CARA FELIZ *(decoraciГіn)*
| Letra | Procedimientos | Tipo de trayectoria |
|-------|---------------|-------------------|
| O | `Path_O_1`, `Path_O_2` | Se reprensenta los ojos de la figura dos lineas |
| B | `Path_B_1`, `Path_B_2` | LГӯnea + arco representando la boca de la figura |
| C grande | `Path_C_1`, `Path_C_2` | Dos arcos MoveC (cГӯrculo completo) repesenta el contorno de la cara feliz|
| C pequeГұo | `Path_C_3`, `Path_C_4` | Dos arcos MoveC (cГӯrculo completo) repesenta el contorno de la cara feliz|

### Home

| Procedimiento | DescripciГіn |
|--------------|-------------|
| `Path_Home_b` | Lleva el robot a la posiciГіn segura de transiciГіn entre letras (z = -100 en el WObj). Usa `MoveL` a v200, z10. |
| `Path_Home` | Lleva el robot a la posiciГіn home global con `tool0` y `wobj0`. |
